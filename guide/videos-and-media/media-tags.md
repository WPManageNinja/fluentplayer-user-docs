# Media Tags (Pro)

Tags let you organize your media library with simple labels like `tutorial`, `onboarding`, or `season-1`. Once your media is tagged, you can filter the library by tag and build **dynamic [tag-based playlists](/guide/playlists/tag-based-playlists)** that update themselves as you tag more videos.

::: tip Pro feature
Media tags require **Fluent Player Pro**. With the free plugin you can still organize media by title and provider, but tagging and tag-based playlists are Pro only.
:::

## Where Tags Live

Tags are managed entirely inside the Fluent Player dashboard, not in the standard WordPress **Posts → Tags** screen. Each tag is stored against your media items so it can be reused across the library and across playlists.

## Adding Tags to a Media Item

1. Go to **Fluent Player → Media** and open the media item you want to tag.
2. Find the **Tags** field in the media settings.
3. Type a tag name and press Enter to add it. Start typing an existing tag to reuse it, or type a new name to create one on the fly.
4. Add as many tags as you need, then **Save**.

A single media item can carry multiple tags, and the same tag can be shared by any number of media items.

## Filtering the Media Library by Tag

The admin media list includes a **tag filter** alongside the provider and media-type filters. Pick one or more tags to narrow the list to just the matching media. This is the fastest way to find every video in a series or topic.

## Managing Your Tags

From the dashboard you can:

- **Create** a new tag.
- **Rename** an existing tag — the new name applies everywhere that tag is used.
- **Delete** a tag — this removes the label from your media but never deletes the media itself.

Tag names must be unique. If you try to create or rename a tag to a name that already exists, Fluent Player keeps the existing one to avoid duplicates.

## Using Tags in Playlists

The real power of tags is dynamic playlists. Instead of hand-picking videos, you can render a playlist from one or more tags — and any media you tag later is included automatically. See [Tag-Based Playlists](/guide/playlists/tag-based-playlists) for the full guide.
